Lab 10 Readme
Який корпус використано

Evaluation set: 20 коротких текстів з ручною розміткою (gold set) включно з повторними фразами та прикладами доменних сутностей (військова техніка, абревіатури, валюти, дати).

Який pipeline запущено

Baseline: spaCy uk_core_news_sm (стандартний NER).

Hybrid: та сама модель + EntityRuler (словники) + regex‑based component (char‑level).

Які rules додано

EntityRuler: словникові патерни для PRODUCT (су‑34, mh17, сапфір, рятувальне судно) і TITLE (генпрокурор, міністр, президент тощо).

Regex component: char‑level правила для MONEY (грн, $, €; діапазони, тисячні роздільники), PERCENT (50‑60%), QUANTITY/DISTANCE (100+ км, 100 000, м), DATE (день+місяць, роки), multiword PRODUCT.

Що baseline знаходив добре

LOC у більшості випадків; деякі PER також коректно розпізнавались. Baseline давав відносно високу точність для простих географічних імен.

Що baseline пропускав

Домени: PRODUCT, MONEY, QUANTITY, PERCENT, DATE, TITLE — майже всі приклади цих класів були FN у baseline.

Що rules покращили

Значне зменшення FN і збільшення TP для доменних класів. TP піднявся з 33 → 52, FN знизився з 33 → 14, Precision підвищилась до 0.912, Recall до 0.788, F1 до 0.846. Доменні класи отримали повне покриття у hybrid.

Які проблеми ще не вирішені

Абревіатури ORG (НАТО, ЄС, ДНР, ППО) — потребують словникових патернів або контекстних правил.

Персональні імена у відмінках — додати варіанти або словник прізвищ.

Span boundary issues — часткові FP і злиття span (потрібен post‑processing).

Довгостроково: розглянути донавчання моделі на розширеному корпусі з доменними анотаціями для стабільнішого розпізнавання складних ORG і відмінкових форм.
