# NER Notes Lab 10



Pipeline вибрано  

uk\_core\_news\_sm (spaCy) + EntityRuler (словникові патерни) + regex‑based component (char‑level для багатотокенних шаблонів).



Сутності, які модель уміє з коробки



LOC, ORG, PER — базова модель розпізнає географічні назви, організації та персональні імена з помірною точністю.



Інші класи (PRODUCT, MONEY, QUANTITY, PERCENT, DATE, TITLE) — мінімальне або відсутнє покриття у baseline.



Доменні сутності важливі для корпусу



PRODUCT (су‑34, MH17, рятувальні судна тощо)



QUANTITY / DISTANCE (100+ км, 100 000)



MONEY / PERCENT (20‑25 грн., 50‑60%)



DATE (11 вересня, дні/місяці, роки)



TITLE (генпрокурор, міністр, очільник)



Абревіатури ORG (НАТО, ЄС, ДНР, ППО)



Правила, які додано



EntityRuler: словникові патерни для PRODUCT і TITLE (су‑34, mh17, сапфір, генпрокурор, міністр тощо).



Regex component (char‑level): правила для MONEY (грн, $, €; діапазони, тисячні роздільники), PERCENT (50‑60%), QUANTITY (км, м, 100+ км, 100 000), DATE (день+місяць, роки), multiword PRODUCT (рятувальне судно "сапфір").



Що стало краще після правил



TP зріс з 33 → 52; FN зменшився з 33 → 14.



Precision піднялась з 0.825 → 0.912, Recall з 0.50 → 0.788, F1 з 0.623 → 0.846.



Повністю покриті доменні класи: PRODUCT, QUANTITY, PERCENT, DATE, MONEY, TITLE (recall = 1.0 для цих класів у hybrid).



Які помилки залишилися



ORG: часті FN і часткові FP для складних назв (наприклад, «5‑ї служби фсб» розбивається на частини).



Абревіатури: «НАТО», «ЄС», «ДНР», «ППО» — FN або неправильна категорія (LOC vs ORG).



PER: деякі прізвища (Пелосі, Кадирова, Лаврова) пропускаються у відмінкових формах.



Span errors: злиття суміжних токенів у один ент (наприклад «лаврова єс») або виділення лише частини ORG (FP: «5‑ї служби», «фсб»).



Що фіксити далі



Розширити словники ORG і PER (акроніми, часті прізвища, бренди) з урахуванням відмінків.



Post‑processing: фільтр вкладених/коротких ентів, корекція злиттів span (розділяти «PERSON + ORG»).



Звузити/відфільтрувати regex у разі появи FP; додати контекстні умови (наприклад, валюта поруч із числом для MONEY).



Зібрати додаткові приклади для ORG/PER у різних відмінках і додати до правил або до тренування моделі.
